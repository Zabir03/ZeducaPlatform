Introduction to Computer Organization

Computer Organization refers to the operational units and their interconnections that realize the architectural specifications. It describes how the hardware components are structured to form a computer system.

Organization vs. Architecture

  • Architecture: The "what." It's the functional behavior of the system, visible to a programmer (e.g., instruction set, data types).
  • Organization: The "how." It's the physical implementation of the architecture (e.g., control signals, memory technology, CPU design).

Basic Components of a Computer

A computer system is fundamentally composed of three main parts that work together to execute programs.

  • CPU (Central Processing Unit): The brain of the computer. It fetches instructions, decodes them, and executes them. Contains the ALU (Arithmetic Logic Unit) and Control Unit.
  • Main Memory (RAM): Stores data and instructions that are actively being used by the CPU. It is volatile storage.
  • I/O Devices: Allow the computer to interact with the outside world. Includes input (keyboard, mouse) and output (monitor, printer) devices, as well as storage (HDD, SSD).

Instruction Set Architecture (ISA)

The ISA is the interface between hardware and software. It defines the set of all instructions that a processor can execute, including instruction formats, addressing modes, and registers.

; Example of a simple instruction
MOV AX, 5   ; Move the value 5 into the AX register
ADD AX, 10  ; Add 10 to the value in the AX register

Memory Hierarchy

A computer's memory is organized in a hierarchy to balance speed, cost, and capacity. The closer the memory is to the CPU, the faster and smaller it is.

  • Registers: Fastest, smallest, inside the CPU.
  • Cache Memory: Fast buffer between CPU and main memory.
  • Main Memory (RAM): Larger and slower than cache.
  • Secondary Storage: Slowest, largest, non-volatile (SSDs, HDDs).

Pipelining

A technique where multiple instructions are overlapped in execution, similar to an assembly line. It increases the overall instruction throughput.

Stages of a Pipeline

A typical pipeline breaks instruction execution into stages like: Fetch, Decode, Execute, and Write Back. While one instruction is executing, the next one is being decoded, and so on.

I/O Systems

Manages the communication between the CPU/memory and peripheral devices.

Methods of I/O Transfer

  • Programmed I/O: The CPU continuously checks the device's status. Inefficient.
  • Interrupt-driven I/O: The device sends an interrupt signal to the CPU when it's ready. More efficient.
  • Direct Memory Access (DMA): A DMA controller handles data transfer directly between I/O devices and memory, freeing up the CPU. Most efficient for large data transfers.
