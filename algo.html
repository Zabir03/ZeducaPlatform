Kadane's Algorithm

The ultimate solution for finding the largest sum of a contiguous subarray in an array of numbers.

The Core Idea

Iterate through the array, keeping track of two values: the maximum sum ending at the current position (`current_max`) and the maximum sum found so far (`global_max`). For each element, you decide whether to extend the current subarray or start a new one.

function kadanes(arr):
  current_max = 0
  global_max = -Infinity

  for each number in arr:
    current_max = max(number, current_max + number)
    if current_max > global_max:
      global_max = current_max
      
  return global_max

Video Explanation

  • Time Complexity: O(n) - Single pass through the array.
  • Space Complexity: O(1) - Only a few variables are used.

Moore's Voting Algorithm

A highly efficient algorithm to find the majority element in an array, i.e., the element that appears more than n/2 times.

The Core Idea

The algorithm works on the principle of cancellation. It maintains a `candidate` element and a `count`. It iterates through the array; if the current element matches the candidate, the count is incremented. If it doesn't, the count is decremented. If the count reaches zero, a new candidate is chosen. The element that remains as the candidate at the end is the potential majority element (which must then be verified in a second pass).

function mooresVoting(arr):
  candidate = null
  count = 0

  for each number in arr:
    if count == 0:
      candidate = number
    count += (1 if number == candidate else -1)

  count = 0
  for each number in arr:
    if number == candidate:
      count++
  
  return candidate if count > length(arr)/2 else null

Video Explanation

  • Time Complexity: O(n) - Two passes at most.
  • Space Complexity: O(1) - Constant extra space.

Sorting Algorithms

Sorting is the process of arranging items in a specific order. Different algorithms offer trade-offs between speed, simplicity, and memory usage.

Common Algorithms

Bubble Sort

Repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order. The pass through the list is repeated until the list is sorted. Simple but inefficient (O(nÂ²)).

function bubbleSort(arr):
  n = length(arr)
  for i from 0 to n-1:
    swapped = false
    for j from 0 to n-i-2:
      if arr[j] > arr[j+1]:
        swap(arr[j], arr[j+1])
        swapped = true
    if not swapped:
      break

Merge Sort

A divide-and-conquer algorithm. It divides the array into halves, recursively sorts them, and then merges them back together. Efficient and stable (O(n log n)).

function mergeSort(arr):
  if length(arr) <= 1:
    return arr
  
  mid = length(arr) / 2
  left_half = mergeSort(arr[0 to mid])
  right_half = mergeSort(arr[mid to end])
  
  return merge(left_half, right_half)

function merge(left, right):
  result = []
  while left is not empty and right is not empty:
    if left[0] < right[0]:
      append left[0] to result
      remove first element from left
    else:
      append right[0] to result
      remove first element from right
  
  append remaining elements from left or right to result
  return result

Quick Sort

Another divide-and-conquer algorithm. It picks a 'pivot' element and partitions the other elements into two sub-arrays, according to whether they are less than or greater than the pivot. Highly efficient on average (O(n log n)).

function quickSort(arr, low, high):
  if low < high:
    pi = partition(arr, low, high)
    quickSort(arr, low, pi - 1)
    quickSort(arr, pi + 1, high)

function partition(arr, low, high):
  pivot = arr[high]
  i = low - 1
  for j from low to high - 1:
    if arr[j] < pivot:
      i++
      swap(arr[i], arr[j])
  swap(arr[i+1], arr[high])
  return i + 1

Searching Algorithms

Searching algorithms are designed to find an element within a data structure.

Linear Search

Sequentially checks each element of the list until a match is found or the whole list has been searched. It can be used on unsorted lists. Simple but slow (O(n)).

function linearSearch(arr, target):
  for i from 0 to length(arr)-1:
    if arr[i] == target:
      return i
  return -1

Binary Search

Efficiently finds an item from a sorted list by repeatedly dividing the search interval in half. Very fast (O(log n)).

function binarySearch(arr, target):
  low = 0
  high = length(arr) - 1

  while low <= high:
    mid = floor((low + high) / 2)
    if arr[mid] == target:
      return mid
    else if arr[mid] < target:
      low = mid + 1
    else:
      high = mid - 1
      
  return -1

Two-Pointer Technique

An elegant and efficient technique that uses two pointers to iterate over a data structure, typically a sorted array, until they meet.

Example: Find a Pair with a Given Sum in a Sorted Array

Place one pointer (`left`) at the beginning and another (`right`) at the end. If their sum is the target, you're done. If the sum is too small, move the `left` pointer forward to increase the sum. If it's too large, move the `right` pointer backward to decrease the sum.

function findPair(arr, targetSum):
  left = 0
  right = length(arr) - 1
  
  while left < right:
    currentSum = arr[left] + arr[right]
    if currentSum == targetSum:
      return (arr[left], arr[right])
    else if currentSum < targetSum:
      left++
    else:
      right--
      
  return null

Video Explanation

  • Time Complexity: O(n) - Because the pointers only move in one direction.
  • Space Complexity: O(1)

Greedy Approach

A problem-solving heuristic of making the locally optimal choice at each stage with the hope of finding a global optimum. It doesn't always work, but is very powerful for certain problems.

Example: Activity Selection Problem

Given a set of activities with start and end times, select the maximum number of non-overlapping activities. The greedy choice is to always pick the next activity whose finish time is earliest among the remaining activities that don't overlap with the previously selected one.

function activitySelection(activities):
  sort(activities, by finish_time)
  
  selected_count = 1
  last_finish_time = activities[0].finish_time
  
  for i from 1 to length(activities)-1:
    if activities[i].start_time >= last_finish_time:
      selected_count++
      last_finish_time = activities[i].finish_time
      
  return selected_count

Sliding Window Technique

This technique is used to efficiently solve problems that involve a contiguous subarray or substring. A "window" of a certain size slides over the data, and we only operate on the data within that window.

Example: Max Sum Subarray of Size K

Instead of recalculating the sum for every subarray, you can calculate the sum of the first window. Then, for each step, you slide the window by one element: subtract the element that leaves the window and add the new element that enters. This reduces the complexity from O(n*k) to O(n).

function maxSumSubarray(arr, k):
  if length(arr) < k:
    return -1
    
  window_sum = sum of first k elements
  max_sum = window_sum
  
  for i from k to length(arr)-1:
    window_sum = window_sum - arr[i-k] + arr[i]
    max_sum = max(max_sum, window_sum)
    
  return max_sum

Slow & Fast Pointer

A powerful technique for LinkedLists, also known as Floyd's Tortoise and Hare algorithm. It uses two pointers that move at different speeds.

Use Case 1: Cycle Detection

Move one pointer by one step (`slow`) and the other by two steps (`fast`). If the pointers ever meet, there is a cycle in the list.

function hasCycle(head):
  slow = head
  fast = head
  
  while fast is not null and fast.next is not null:
    slow = slow.next
    fast = fast.next.next
    if slow == fast:
      return true
      
  return false

Use Case 2: Finding Middle Element

When the `fast` pointer reaches the end of the list, the `slow` pointer will be at the middle.

function findMiddle(head):
  slow = head
  fast = head
  
  while fast is not null and fast.next is not null:
    slow = slow.next
    fast = fast.next.next
    
  return slow

Reverse a LinkedList

A common interview problem that involves reversing the direction of pointers in a singly linked list.

The Core Idea

You need three pointers: `prev` (to store the previous node), `current` (the node you're currently visiting), and `next_node` (to store the next node before you change the pointer). Iterate through the list, and for each node, store the `next_node`, reverse the `current` node's pointer to `prev`, and then move `prev` and `current` one step forward.

function reverseList(head):
  prev = null
  current = head
  
  while current is not null:
    next_node = current.next
    current.next = prev
    prev = current
    current = next_node
    
  return prev
