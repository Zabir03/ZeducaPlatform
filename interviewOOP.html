The Complete Guide to OOPs

Object-Oriented Programming concepts for placement interviews.

1. Introduction to OOP

Definition: OOP is a programming paradigm based on objects which encapsulate data (attributes) and behavior (methods).

Core concepts:

  • Class & Object
  • Encapsulation
  • Abstraction
  • Inheritance
  • Polymorphism

2. Class and Object

  • Class: Blueprint/template that defines attributes and methods.
  • Object: Instance of a class.

Example in Java:

class Car {
    String color;
    int speed;
    void drive() {
        System.out.println("Car is driving...");
    }
}

public class Test {
    public static void main(String[] args) {
        Car c1 = new Car(); // Object creation
        c1.color = "Red";
        c1.speed = 100;
        c1.drive();
    }
}

3. Encapsulation

Wrapping data (variables) and code (methods) together. It protects data using access modifiers and provides getters and setters.

class Student {
    private String name;
    public void setName(String n) { name = n; }
    public String getName() { return name; }
}

  • What is encapsulation and how does it improve security?

4. Abstraction

Hiding implementation details while showing only functionality. Achieved using abstract classes or interfaces.

// Abstract Class Example
abstract class Shape {
    abstract void draw();
}

// Interface Example
interface Drawable {
    void draw();
}

5. Inheritance

Mechanism to re-use code by acquiring properties from another class.

class Vehicle {
    void start() { /* ... */ }
}
class Car extends Vehicle {
    // Car inherits start() method
}

6. Polymorphism

The ability of an object or method to take many forms.

  • Compile-time (Method Overloading): Same method name, different parameters.
  • Run-time (Method Overriding): Child class redefines parent method.

7. Constructors and Destructors

  • Constructor: Special method to initialize objects; same name as class.
  • Destructor: Cleans up memory. Automatic in Java (Garbage Collection), explicit in C++ (~ClassName()).
